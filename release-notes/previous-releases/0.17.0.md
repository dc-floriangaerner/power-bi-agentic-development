# v0.17.0

## New Skills

- Created a `power-query` skill in the `semantic-models` plugin with basic guidance on writing M expressions for semantic model partitions and how to execute Power Query using [`pq-cli`](https://github.com/data-goblin/pq-cli) or the Fabric executeQuery API which `pq-cli` wraps. Includes query folding guidance with a comprehensive fold-breaker list (~80 operations), two validation approaches (executeQuery API and XMLA save), step-by-step debugging, and best practices reference.
- **PQTest reference**: documents local Power Query execution via Microsoft's PQTest.exe (Windows only), mashup engine architecture, and comparison with the Fabric executeQuery API.
- Example Python scripts (`execute_m.py`, `preview_partition.py`) for executing M expressions and previewing partition data at any transformation step.

## Improvements

- Updated README badges to Power BI, Microsoft Fabric, and Tabular Editor.
- Added a [`useful-stuff`](../useful-stuff/) folder with defensive hooks for general agent safety (block npm/pip/destructive commands/secrets exposure). All hooks use command-position regex matching to avoid false positives.
- Added support for local model files (TMDL, BIM) to the `connect-pbid` skill.
- Updated the Fabric CLI skill to document how to execute Python or PySpark ephemerally on Fabric runtime (thanks [Alex Dupler](https://x.com/alexdupler)).
- Added Fabric Connections reference to the Fabric CLI skill (create, update, list connections; credential types).
- Enhanced DuckDB reference to better clarify approach when lakehouse schemas are present.
- Updated notebook CLI references for `nb exec code` and `nb exec cell` subcommands.

## Known Limitations

- The `executing-power-query` skill (ETL plugin) is not included in this release; it will ship when the ETL plugin is ready.
- PQTest.exe is Windows-only (.NET Framework 4.7.2); macOS/Linux users should use the Fabric executeQuery API for M execution.
